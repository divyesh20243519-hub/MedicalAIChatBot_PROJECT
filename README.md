# Medical Vision AI Assistant
This project is a web application built with FastAPI that allows users to upload medical images and ask questions about them in natural language. The application sends the image and query to a vision-language model (Qwen3.6) through the Groq API and displays the generated response.
The project is intended for simple educational and informational medical queries. It is **not** a substitute for professional medical advice, diagnosis, or treatment.
---
## Features
* Upload a medical image
* Ask questions in natural language
* Receive a response from the Qwen3.6 vision-language model
* Validate uploaded images before processing
* Handle invalid inputs and API errors gracefully
---
## Technologies Used
* Python
* FastAPI
* Jinja2
* Uvicorn
* Pillow
* Requests
* python-dotenv
* Groq API
---
## How It Works
1. The user uploads a medical image and enters a question.
2. The image is validated using Pillow.
3. The image is encoded into Base64 format.
4. The image and prompt are sent to the Qwen3.6 vision model through the Groq API.
5. The response is returned to the frontend.
6. The user can view the generated answer on the webpage.
---
## Notes
This project was built to explore the use of vision-language models in answering simple questions about medical images while learning FastAPI, API integration, image processing, and backend development. The responses generated by the model are intended for educational purposes only and should not be relied upon for medical decisions.
